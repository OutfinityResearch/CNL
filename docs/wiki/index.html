Wiki

Shared language and working conventions.

This wiki defines the vocabulary and internal conventions that keep CNL-PL consistent across specs, tooling, and evaluation.

Quick definitions

  • CNL-PL: Controlled Natural Language Programming Language.
  • AST: The canonical, lossless parse tree of a sentence.
  • Pragmatics: The execution mode applied to a parsed sentence.

Key terms

Assertion

The atomic statement form: subject, verb, object or controlled copula.

Condition

A boolean structure built from assertions, with explicit grouping rules.

Relative clause

A constraint attached to a noun phrase using explicit pronouns.

Action block

Multi-line structure describing actions, preconditions, and effects.

Project conventions

  • All specs and documentation are written in English.
  • Design specifications are stored as DSxx markdown files.
  • AST shapes are lossless and preserve lexical spans.
  • Tests and evaluation suites map to runtime components.
