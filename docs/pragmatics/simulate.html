Simulate

Run state transitions over time to model future states.

Concept

Simulate is used to see "what happens next". It applies transition rules iteratively for N steps. Unlike "Plan" (which seeks a goal), Simulate simply evolves the system based on its current physics/rules.

Syntax

Simulate for 10 steps.

Execution Logic

The engine identifies all Active Actions (actions whose preconditions are currently met).

  • Step 1: Identify enabled actions (e.g., "Timer Ticks", "Process Queue").
  • Step 2: Apply effects (update bitsets).
  • Step 3: Re-evaluate preconditions for the next step.

This turns the KB into a State Machine. The output is a trace of state changes over time.
