CNL Pragmatics
Pragmatic Mode

Solve

Resolve constraints and find valid variable assignments.

Concept

The Solve pragmatic treats the CNL input as a Constraint Satisfaction Problem (CSP). It is used when you don't know who or what fits, but you know the rules they must follow.

Syntax

Solve for every schedule such that every task is assigned to a user
and every user has the skill for the task
and no user has more than 5 tasks.

Execution Logic

The compiler identifies the variables (schedule, task, user) and the constraints.

1. Domain Definition: user is the set of all users in KB. task is the set of tasks.
2. Constraint Propagation: "user has skill" filters the domain of possible assignments.
3. Solver Backend: The problem is often handed off to an SMT solver (like Z3) or a backtracking solver to find a valid assignment matrix.
