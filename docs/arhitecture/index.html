Architecture

From plain text to executable meaning.

The CNL-PL stack follows a deterministic pipeline: lexical analysis, parsing, validation, AST normalization, and pragmatic execution.

Pipeline stages

  • Lexer: tokens with longest-match keyword rules.
  • Parser: AST construction with explicit grouping.
  • Validator: grammar errors, mixed operators, missing terms.
  • Pragmatics: query, proof, plan, solve, simulate, explain.

Subsystems

Grammar + AST

Formal EBNF with a lossless AST specification for tooling and serialization.

Validation layer

Detects mixed boolean operators, missing terminators, and noun phrase errors.

Pragmatics engine

Interprets AST nodes against semantic backends or knowledge stores.

Execution adapters

Connects AST to proof systems, solvers, planners, and data stores.
